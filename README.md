# 图像侵权风险自动比对工具

这是一个纯前端、本地运行的中文网页工具，用于做设计图的内部侵权初筛。

它不依赖 OpenAI、后端接口或 API Key，所有图片处理和自动比对都在浏览器本地完成。

## 当前能力

- 上传设计图后，自动与本地高风险图库逐张比对
- 支持上传一张指定对比图做补充参考
- 支持手动添加高风险对象，并给对象填写分类、备注、标签
- 支持按文件夹批量导入图库图片
- 支持图库搜索、分类筛选、标签筛选
- 输出 Top 5 命中结果
- 输出规则判定依据、风险关注点和修改建议
- 支持在页面内调整高风险 / 中风险阈值，并保存在当前浏览器
- 支持导出图库 JSON 和分析报告 TXT

## 当前判定逻辑

系统会先做自动图像相似度计算，再把结果映射到你要求的规则：

1. 三项核心维度规则

- 0 项不同 = 高风险
- 1 项不同 = 高风险
- 2 项不同 = 中风险
- 3 项不同 = 低风险

2. 单一图案规则

- 主体没变且视觉接近 = 高风险
- 主体没变但视觉不同 = 中风险
- 主体变了且视觉不同 = 低风险

3. 组合图案规则

- 差异 < 50% = 高风险
- 差异 >= 50% 但视觉接近 = 中风险
- 差异 >= 50% 且视觉不同 = 低风险

4. 一票高风险规则

- 品牌 / IP 命中 = 高风险
- AI 保留原轮廓 = 高风险

最终结果采用“取最高风险规则”的方式输出。

## 自动判断是怎么做的

浏览器会从图片中提取这些特征：

- 颜色分布
- 主体轮廓
- 构图结构
- 边缘组织
- 平均哈希
- 差异哈希

并据此生成：

- 主体接近度
- 构图接近度
- 视觉接近度
- 整体接近度

系统还会自动估计图案更像“单一图案”还是“组合图案”，再套用对应规则。

## 批量导入建议

推荐把每个风险对象单独放在一个文件夹中，例如：

- `Disney/米奇/图1.png`
- `Disney/米奇/图2.png`
- `Sanrio/Hello Kitty/图1.png`

系统会优先使用：

- 最后一层文件夹名作为风险对象名称
- 上一层文件夹名作为分类

## 部署

这是一个静态站点，可以直接部署到：

- [Vercel](https://vercel.com/)
- [Netlify](https://www.netlify.com/)
- Cloudflare Pages

把整个文件夹上传即可，不需要后端。

## 说明

- 这是内部初筛工具，不等同于最终法律判断
- 图库越完整，自动比对结果越有参考价值
- 如果你后续还想继续加强，可以再加两类能力：
  - 更大的品牌 / IP 图库
  - 本地模型或第三方视觉识别服务
